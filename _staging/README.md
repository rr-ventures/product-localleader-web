# Local Leader Staging Pack

This folder is the clean staging area for planning and build agents.

## Start Here

Read `build-guide.md` -- it is the single reference document containing page structure, copy direction, design constraints, CTA strategy, and the complete asset inventory with every graphic and logo described.

## Folder Structure

- `build-guide.md` -- Implementation guide + asset inventory (combined).
- `graphics/` -- Flat bucket of 38 unique proof graphics, explainers, hero visuals, and generated illustrations.
- `logos/` -- Flat bucket of 6 unique logo options.
- `references/v3/` through `references/v5/` -- Competitor reference screenshots for new version designs.
- `references/v1/` and `references/v2/` -- Placeholders for future aesthetic change references.
- `archived-asset-planning-notes.md` -- Previous version-specific planning notes preserved for fallback context only.

## Guidance For Next Agents

- Treat `graphics/` and `logos/` as shared buckets.
- Do not assume an asset is locked to a specific version.
- Review the screenshots in `references/` first, then choose the most suitable assets from the shared buckets.
- Use `archived-asset-planning-notes.md` only if you want to revisit the older prescriptive approach.
